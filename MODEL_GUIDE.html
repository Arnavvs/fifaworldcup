1. The Big Picture

Think of our prediction system as a football analysis pipeline with 5 stages:

1. Data Collection
49,000+ historical matches, player stats, ELO ratings, weather, travel
→
2. Feature Engineering
Turn raw data into 55 "features" the models can understand
→
3. Model Zoo
4 different models each make their own prediction
4. Stacking Ensemble
Combine the 4 models into one "super prediction"
→
5. Simulation
Run 50,000 virtual tournaments to get champion odds
→
6. Dashboard
Display probabilities, charts, chaos meter
Key idea: No single model is perfect. We use multiple models with different strengths, then combine them. It's like asking 4 experts for their opinion and averaging the answers — but weighted by how good each expert has been historically.

4. Model 1: ELO + Davidson (The "Common Sense" Baseline)

Model 1 Baseline Test LL: 0.868

What it does

Every team has a rating (like chess ELO). After each match, ratings update based on result. The model predicts the next match using only these ratings.

The Math (simplified)

# ELO update formula new_rating = old_rating + K × G × (actual_result - expected_result) # Where: # K = importance (60 for World Cup, 20 for friendlies) # G = goal margin factor (1 for close games, up to 3 for blowouts) # expected_result = 1 / (1 + 10^(-rating_difference/400))

Davidson Draw Model

Standard ELO only predicts win/loss. We add the Davidson model to estimate draws:

# Davidson draw probability pi = 10 ^ ((home_rating - away_rating + home_advantage) / 400) draw_prob = nu × sqrt(pi) / (pi + 1 + nu × sqrt(pi)) # nu = draw parameter (fitted: 0.705) # home_advantage = 142 points (fitted from data)

Key Parameters

ParameterValueMeaning
K_factor60 (WC), 50 (continental), 40 (qualifiers), 20 (friendlies)How fast ratings change
nu0.705Draw tendency (higher = more draws)
home_adv142Home advantage in ELO points
initial_rating1500Starting ELO for new teams

5. Model 4: Dixon-Coles (The "Goals Engine")

Model 4 Goals Model Test LL: 0.914

What it does

Instead of predicting win/draw/loss directly, this model predicts how many goals each team scores. Then it derives win/draw/loss from the score distribution.

Why this matters: The simulator needs to sample scores (e.g., "2-1" or "0-0"), not just outcomes. Dixon-Coles is the gold standard in football analytics.

The Math

# Expected goals = Poisson with team attack/defense strengths home_goals ~ Poisson(exp(μ + home_adv + attack_home - defense_away)) away_goals ~ Poisson(exp(μ + attack_away - defense_home)) # Dixon-Coles correction for low scores (0-0, 1-0, 0-1, 1-1) tau = 1 - rho × lambda_home × lambda_away # for 0-0 # rho = -0.01 (fitted — negative means fewer 0-0s than expected)

Key Parameters

ParameterValueMeaning
mu0.035Average log-goals baseline
home_adv0.262Home advantage in log-goals
rho-0.01Low-score correlation (Dixon-Coles parameter)
xi0.0019/dayTime decay (1-year half-life)
min_matches30Minimum team appearances to get own parameter

Time Decay

Recent matches matter more. A match from 2020 gets weight exp(-0.0019 × days_ago). A match from 2026 gets weight 1.0.

6. Model 6: The Stacking Ensemble (The "Brain")

Model 6 Meta-Learner Best: 0.857

What it does

We train 4 "level-0" models, then teach a 5th model (the "meta-learner") how to best combine them. It's like a portfolio manager who decides how much to invest in each stock.

Level-0 Models

1. Davidson (ELO) — Ratings-based

2. Logistic Regression — Linear, interpretable

3. Histogram GBM — Gradient boosting trees

4. Dixon-Coles — Goals-based

Meta-Learner

Multinomial Logistic Regression on logit(probs)

Inputs: each model's 3 probabilities + context flags

Output: final blended probability [loss, draw, win]

How We Prevent Cheating (Out-of-Fold)

The meta-learner can't see the actual predictions on the matches it will predict — that would be cheating. Instead, we use 5 expanding-window folds:

# Fold 1: train 1872-1995, predict 1995-2000 # Fold 2: train 1872-2000, predict 2000-2005 # Fold 3: train 1872-2005, predict 2005-2010 # Fold 4: train 1872-2010, predict 2010-2015 # Fold 5: train 1872-2015, predict 2015-2018 # → Meta-learner trains on these 5 "honest" prediction sets

Temperature Calibration

The raw ensemble is slightly overconfident. We apply temperature scaling (T=1.099) to soften probabilities:

soft_prob = softmax(logit(raw_prob) / T) # T > 1.0 = more uncertain (better calibrated) # Fitted on validation set to minimize log-loss

7. Model 8: Tournament Simulator (The "Monte Carlo" Crystal Ball)

Model 8 Simulator 50,000 runs

What it does

We don't just predict one match — we simulate the entire tournament 50,000 times. Each simulation is a parallel universe where results are random but weighted by our probabilities.

Monte Carlo = "What if we played the World Cup 50,000 times?" In each run, every match gets a random score drawn from the Dixon-Coles probability distribution. We aggregate all 50,000 universes to see how often Argentina wins, how often Spain reaches the final, etc.

Simulation Steps

1. Group Stage
72 matches, 12 groups of 4. Sample scores from DC matrix.
2. Rank Groups
Points → Goal Difference → Goals → Random tiebreak.
3. Best Thirds
8 of 12 third-place teams advance. Rank by points/GD/goals.
4. Knockout
R32 (16 matches) → R16 (8) → QF (4) → SF (2) → Final
5. Knockout Draws
90' draw → ET (Poisson λ/3) → still draw → Penalties

Penalty Model (from real data!)

We fitted a penalty shootout model from 572 real shootouts:

p_higher_elo_wins = sigmoid(0.0185 + 0.6265 × |elo_diff| / 400) # Example: 400 ELO gap → higher team wins 65.6% of shootouts # Example: 0 ELO gap → 50.5% (basically coin flip)

8. Model 9: Entropy Engine (The "Chaos Meter")

Model 9 Entropy Chaos

What it does

Measures how surprising the tournament is. If upsets happen, chaos is high. If favorites win, chaos is low.

Three Key Metrics

H = Pre-Match Entropy

H = -Σ p_i × ln(p_i)

How uncertain was the match before it started?

50/50 match → H = 1.10 (max)

90/10 match → H = 0.33 (boring)

I = Realized Surprisal

I = -ln(p_of_actual_result)

How shocking was the actual result?

Favorite wins → I = 0.3-0.7

Major upset → I = 2.0+

X = Excess Surprisal

X = I - H

Was the result more chaotic than expected?

X > 0 → more chaos than probabilities predicted

9. Model 10: Golden Boot Scorer Predictions

Model 10 Scorers Top 3

What it does

Predicts who will win the Golden Boot (most goals). For each player, we estimate their share of their team's goals, then multiply by the team's expected tournament goals.

The Formula (per player)

expected_goals = team_expected_goals × goal_share goal_share = ( shrunk_career_rate × minutes_share + penalty_bonus ) / team_total # shrunk_career_rate = (goals/caps) shrunk toward position average # minutes_share = 1.0 for starters, 0.35 for bench # penalty_bonus = +0.08 if player takes >50% of team penalties

11. The Live Loop (How Predictions Update Every Matchday)

Automated Pipeline

1. Ingest Scores
Check fixturedownload.com + Wikipedia for new results
→
2. Update Database
Store scores in matches + wc2026_fixtures
→
3. Update ELO
Recompute ratings for both teams (K=60)
4. Re-simulate
Run 50,000 Monte Carlo tournaments with new ELOs
→
5. Compute Surprisal
How shocking was the result? (-ln p)
→
6. Update Dashboard
Write new JSON files, refresh charts
Idempotence: Running the loop twice with no new scores does nothing. We track ingested match numbers in artifacts/ingested.json to avoid double-counting.
