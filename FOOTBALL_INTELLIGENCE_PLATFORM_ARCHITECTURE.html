Football Intelligence Platform

Comprehensive Architecture & Implementation Plan — Principal ML Architect & Research Lead Assessment

Synthesized from research corpus, production system blueprints, and existing data asset audit (worldCup project)

Executive Summary

This document provides a complete architectural blueprint for evolving the existing FIFA World Cup prediction pipeline into a production-grade football intelligence platform. The plan spans three maturity levels — MVP, Research-Grade, and Betting-Grade — and maps every feature taxonomy to its raw data dependency, availability status, and engineering effort.

Core finding: The existing dataset (~626k rows, 27 tables) is sufficient to ship a credible MVP within 1–2 weeks. However, the gap to betting-grade edge is defined by four data layers: (1) market odds (calibration & CLV), (2) spatiotemporal event data (xG/xT/OBV), (3) availability feeds (injuries/lineups), and (4) real-time infrastructure (streaming feature store & sub-millisecond inference).

MVP ready: 1–2 weeks Research grade: 8–12 weeks Betting grade: 16–24 weeks

3. Existing Data Asset Analysis

What Already Exists

  • 49,353 match results (1872→2026, all comps)
  • 98,562 team-match feature rows (30 engineered features)
  • 49,281 research-ready samples (39 leakage-safe features, time-split)
  • 18,142 ELO ratings (year-end, 1901→2026)
  • 67,261 FIFA rankings (1992→2024-04)
  • 18,127 player profiles (FIFA game ratings proxy)
  • 1,246 official WC2026 squads (caps, clubs, positions)
  • 900 WC historical matches (1930→2018)
  • 104 WC2026 fixtures + 48 qualified teams
  • 16 venue geocodes (lat/lng + altitude)
  • StatsBomb xG (WC 2018+2022: 255 team rows, 1,757 player rows)
  • Starting lineups (WC 2018+2022: 2,816 rows)
  • 479,440 historical odds rows (2005→2015)
  • 575 canonical team IDs (dim_team normalization)

What Is Missing

  • Recent betting odds (2016→2026) — critical for calibration
  • xG for qualifiers & friendlies (StatsBomb only covers WC 18/22)
  • Live FIFA rankings (stale since April 2024)
  • Per-match ELO (only year-end snapshots)
  • Injuries & suspensions (table empty, source blocked)
  • Club-season player form (minutes, goals, xG 23/24→25/26)
  • Real-time lineups (2026 fixtures are future events)
  • Weather forecasts (not collected)
  • Referee data (100% null)
  • Travel distances (venues geocoded but distances not computed)
  • Spatiotemporal event data (no full PPDA/Field Tilt/xD feeds)
  • Passing network data (no SNA graphs)
  • Manager/tenure data (no structured manager DB)
  • Environmental data (heat, humidity, pitch moisture)

Critical vs. Postponable

Critical (block MVP→Betting):

  • Live FIFA rankings 2024→2026
  • Recent betting odds (2016→2026)
  • Per-match ELO (or self-computed)
  • Official squad aggregates (caps, age, club quality)

Important (Research grade):

  • xG for all matches (not just WC)
  • Injuries & availability
  • Weather & travel computation
  • Referee profiles

Postponable (Tier 3 / Experimental):

  • 25 Hz optical tracking
  • Real-time in-play Poisson
  • GNN embeddings (until xG/lineups ready)
  • Agent-based RoboCup simulations
  • Psychological momentum flows

5. Recommended Architectures

A) MVP Architecture (Weeks 1–3)

Objective: Ship a credible, backtestable WC 2026 prediction system that beats Elo-only baselines.

Data Layer

  • Existing matches + team_match_features
  • Refresh FIFA rankings to 2026
  • Compute travel distances from venues
  • Integrate official squads (caps, age, club)

Feature Layer

  • Tier 1 features only (20 features)
  • ELO diff, FIFA rank diff, form windows
  • H2H, rest, home/neutral, stage weight
  • Travel TDF, altitude

Model Layer

  • Elo baseline — reference log-loss
  • Dixon-Coles — goal simulation engine
  • LightGBM — W/D/L classifier
  • CatBoost — cross-check classifier

Output Layer

  • Calibrated 1X2 probabilities
  • Poisson goal expectations
  • Monte-Carlo tournament sim (50k runs)
  • WC 2026 win probabilities per team
MVP Validation Gate: The LightGBM/CatBoost ensemble must achieve lower log-loss and higher RPS than the Elo baseline on a time-based walk-forward test (train on ≤2018, predict 2022). Only then should the project advance to Tier 2.

B) Research-Grade Architecture (Weeks 4–12)

Objective: Add spatiotemporal signal, player-level granularity, and tactical context to create a publishable/competitive research platform.

Data Additions

  • StatsBomb 360 / Opta Vision event feeds
  • xG for all matches (not just WC)
  • Full historical lineups + minutes played
  • Manager tenure + tactical assignment logs
  • Weather forecasts (OpenWeather API)
  • Referee assignment + card histories

Feature Additions

  • Tier 2 features (20 features)
  • xGD, xT, PPDA, Field Tilt, xD
  • Lineup cohesion (shared minutes proxy)
  • Squad depth + injury severity weights
  • Gower playstyle distance
  • Weather + referee adjustments

Model Additions

  • HIGFormer GNN — player interaction graphs (passing networks as edges, player OBV as nodes)
  • Temporal Transformer — ordered event sequences for possession progression
  • Bayesian Hierarchical Poisson — attack/defense random effects for sparse data

Infrastructure Additions

  • Feast Feature Store (offline S3 + online Redis)
  • Spark/Flink batch jobs for rolling windows
  • Temporal leakage audit pipeline
  • ONNX export for sub-millisecond inference
Research Gate: The GNN + Transformer ensemble must demonstrate measurable log-loss improvement over the MVP GBDT baseline on a held-out tournament (e.g., Euro 2024 or WC 2022). If improvement is <2%, abandon deep learning and invest in data quality instead.

C) Betting-Grade Architecture (Weeks 13–24)

Objective: Extract positive expected value (+EV) from betting markets via real-time calibration, market inefficiency detection, and automated execution.

Data Additions

  • Pinnacle API (real-time WebSocket odds)
  • Multiple bookmaker feeds (for cross-market arb)
  • Injury/availability real-time feeds
  • Lineup confirmation feeds (60-min pre-kickoff)
  • Live in-play event streams (PPDA, possession)

Feature Additions

  • Tier 3 features (selectively)
  • CLV bias, Shin implied probs, odds volatility
  • Underdog sentiment index
  • Live injury severity updates
  • In-play momentum flow (5-min windows)

Model & Execution Layer

  • Level-0 Ensemble: CatBoost + LightGBM + HIGFormer + Dixon-Coles
  • Level-1 Meta-Learner: Calibrated Ridge / shallow NN combining model probs + market probs
  • Calibration: Isotonic regression on validation set
  • EV Engine: Compare calibrated true prob vs Shin-derived market prob
  • Kelly Criterion: Fractional staking (f*/4) with bankroll management
  • Execution: REST API order placement to sharp books

Risk Controls

  • Maximum bet size limits (1–2% bankroll)
  • Market liquidity filters (avoid low-limit props)
  • CLV tracking (verify edge persists to close)
  • Drawdown circuit breakers (>15% bankroll = halt)
Betting-Grade Gate: Before deploying capital, the system must demonstrate positive closing-line value (CLV > 0) over a minimum of 500 bets in a paper-trading phase. No exceptions.

7. Strategic Assessment & Final Recommendations

What Is Missing?

  • Temporal freshness: FIFA rankings are 2 years stale; ELO is year-end only.
  • Market data: No recent odds (2016→2026). Without this, calibration is impossible.
  • Spatiotemporal events: xG exists only for 2 World Cups. No qualifier/friendly coverage.
  • Availability: Injuries table is empty. No suspensions or fitness data.
  • Live infrastructure: No streaming feature store, no real-time odds ingestion.
  • Referees: 100% null. Referee bias is a real factor, especially in cards/corners.
  • Deep features: No passing networks, no tracking data, no tactical embeddings.

What Should Be Collected Next?

Ranked by expected log-loss improvement per hour of work:

  1. Live FIFA rankings (2024→2026) — 2–4 hours, massive lift
  2. Team-name normalization fix — 2–3 hours, prevents silent failures
  3. Recent odds (2016→2026) — 1–2 days, enables market calibration
  4. Per-match ELO or self-computed ELO — 1 day, sharpens strength signal
  5. Weather forecasts for WC 2026 — 2–3 hours, easy context wins
  6. Referee histories (cards, penalties, extra time) — 1 day, medium lift
  7. xG for all matches (Understat scrape) — 2–3 days, high research value
  8. Injuries via Playwright/Transfermarkt — 2–3 days, requires infra

What Can Be Postponed?

  • 25 Hz optical tracking feeds (Tracab / Second Spectrum) — extreme cost, complex, low ROI unless building custom physical models.
  • Real-time in-play Poisson engine — requires sub-second event ingestion and low-latency execution infrastructure. Only relevant for betting-grade deployment.
  • Agent-based simulations (RoboCup / Google Football) — fascinating research direction, but zero near-term predictive value for WC forecasting.
  • Psychological momentum flows — highly noisy, difficult to validate, marginal signal for 1X2 markets. Better suited to in-play trading research.
  • Language barrier indices — clever idea, but effect size on match outcomes is likely negligible vs. tactical and strength features.
  • Transfer market Gini shocks — more relevant for club-league season-long modeling than single-tournament national team prediction.

Biggest Risks

  • Data leakage via temporal boundaries. If rolling features include the match being predicted, backtests will be falsely inflated. Automated leakage tests are mandatory.
  • Anti-bot blocking on richest data sources. FBref and Transfermarkt are the primary paths to xG, injuries, and club form. Without a residential IP or browser automation stack, these remain inaccessible.
  • Overfitting to small tournament samples. A World Cup has only 64 matches. Training on 49k matches and validating on 64 creates high variance. Use walk-forward on multiple tournaments + leagues.
  • Assuming lineups are known pre-kickoff. Official sheets drop ~60 minutes before kickoff. Pre-match models must use projected lineups or probability distributions, not deterministic XIs.
  • Calibrated probabilities without market benchmark. A model with perfect calibration but no odds comparison cannot identify +EV. The market is the ultimate validator.
  • Premature deep learning. GNNs and Transformers need large, high-resolution event datasets to beat GBDTs. With the current data, they will likely underperform and consume disproportionate engineering time.

9. Synthesis & Decision Framework

The research corpus defines a theoretically optimal platform with 300+ features across 18 taxonomies, requiring premium data feeds (StatsBomb 360, Opta Vision, Pinnacle API), heavy compute infrastructure (Spark/Flink, Feast, ONNX, Celery), and cutting-edge deep learning (HIGFormer, Temporal Transformers).

However, the existing project is not starting from zero. It already possesses:

  • A modular, resumable pipeline architecture
  • 49k matches with 30 engineered features
  • 39 research-ready leakage-safe features
  • Official WC 2026 squads and fixtures
  • Canonical team normalization (dim_team)
  • Time-based train/val/test splits

The optimal strategy is therefore staged convergence, not big-bang rebuild:

  1. Week 1–2: Close data freshness gaps (rankings, ELO, travel) and ship MVP baseline.
  2. Week 3–6: Add odds + xG feeds; build Tier 2 features; validate GBDT ensemble.
  3. Week 7–12: If GBDTs beat Elo by >5% log-loss, invest in GNN/Transformer research. If not, invest in more data.
  4. Week 13+: Only pursue betting-grade infrastructure if paper-trading demonstrates positive CLV over 500+ bets.
Core Principle: In football prediction, data quality and temporal freshness almost always beats model complexity. A LightGBM model with live rankings, per-match ELO, and recent odds will outperform a HIGFormer GNN with stale features. Prioritize data over architecture until the data ceiling is reached.
